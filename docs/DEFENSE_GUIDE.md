# ORDA: сценарий демонстрации для защиты

Тема проекта: веб-система регистрации и управления мероприятиями и конференциями.

## 1. Что сказать в начале

ORDA Smart Event System - это веб-платформа для публикации мероприятий, регистрации участников, управления заявками и просмотра статистики. Система поддерживает роли `user` и `admin`, онлайн/офлайн/гибридные мероприятия, календарь, личный кабинет и admin dashboard.

## 2. Как запустить

```powershell
cd C:\Users\adilb\OneDrive\Desktop\Diplom
node server.js
```

Открыть:

```text
http://localhost:3000
```

## 3. Что проверить перед показом

1. MongoDB Atlas: текущий IP добавлен в `Network Access`.
2. В `.env` есть `MONGODB_URI` и `JWT_SECRET`.
3. Сервер показывает:

```text
ORDA Smart Event System: http://localhost:3000
MongoDB connected
```

4. Команда проходит:

```powershell
npm.cmd test
```

## 4. Демонстрационный сценарий

### Шаг 1: Главная страница

Показать:
- бренд ORDA;
- поиск мероприятий;
- категории;
- светлую/темную тему;
- переключение языка RU/KZ/EN.

### Шаг 2: Регистрация пользователя

Создать аккаунт участника:
- имя;
- email;
- пароль.

Если в базе нет администратора, первый пользователь станет `admin`.

### Шаг 3: Вход

Войти через email/password. После входа JWT сохраняется в браузере, интерфейс показывает имя пользователя и доступные разделы.

### Шаг 4: Мероприятия

Показать:
- список мероприятий;
- поиск;
- фильтр по категории;
- фильтр online/offline/hybrid;
- карточку события;
- подробности события.

### Шаг 5: Регистрация на мероприятие

Нажать `Зарегистрироваться`. После этого запись появляется в личном кабинете.

### Шаг 6: Личный кабинет

Показать:
- профиль участника;
- телефон;
- организация;
- город;
- мои регистрации;
- отмена регистрации.

### Шаг 7: Admin Dashboard

Показать:
- создание мероприятия;
- редактирование мероприятия;
- удаление мероприятия;
- статистику;
- заявки участников;
- check-in.

## 5. Что делать, если база не подключилась

Если сайт открывается, но мероприятия не загружаются:

1. Открыть MongoDB Atlas.
2. Перейти в `Network Access`.
3. Добавить текущий IP.
4. Подождать 1-2 минуты.
5. Перезапустить сервер:

```powershell
taskkill /F /IM node.exe
node server.js
```

## 6. Короткое объяснение архитектуры

Backend разделен на слои:

- `config` - настройки и MongoDB connection;
- `models` - Mongoose модели;
- `middleware` - auth, roles, validation, errors;
- `controllers` - бизнес-логика;
- `routes` - API endpoints;
- `utils` - helpers и demo data.

Frontend разделен на:

- `index.html` - структура интерфейса;
- `styles.css` - premium SaaS дизайн;
- `app.js` - основная логика;
- `js/core` - API, i18n, session, theme, toast, format helpers.

## 7. Финальная фраза

Проект решает задачу автоматизации регистрации участников и управления конференциями. Он может использоваться конференц-залами, учебными заведениями и организациями, которые проводят офлайн, онлайн и гибридные мероприятия.
